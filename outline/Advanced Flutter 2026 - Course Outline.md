## แนวคิดหลักของหลักสูตร

ในปี 2026 การพัฒนาแอปพลิเคชันด้วย Flutter ไม่ได้จำกัดอยู่เพียงการสร้างหน้าจอ (UI) ที่สวยงามอีกต่อไป เมื่อแอปพลิเคชันมีขนาดใหญ่ขึ้นและมีความซับซ้อนทางธุรกิจ การรักษามาตรฐานของโค้ด การวางโครงสร้างโปรเจกต์ให้รองรับการขยายตัว (Scalable Architecture) และการจัดการสถานะ (State Management) ที่มีประสิทธิภาพ จึงกลายเป็นหัวใจสำคัญของการพัฒนา

หลักสูตรนี้ออกแบบมาเพื่อยกระดับนักพัฒนา Flutter จากระดับกลางให้ก้าวสู่ระดับผู้เชี่ยวชาญ โดยมุ่งเน้นการแก้ปัญหาคอขวดในการพัฒนาแอปพลิเคชันขนาดใหญ่ ตลอดการอบรมแบบเข้มข้น 3 วัน ผู้เรียนจะได้เจาะลึกการใช้งาน **Riverpod** ซึ่งเป็นเครื่องมือจัดการ State ที่ทันสมัย ปลอดภัย และลดความซ้ำซ้อนของโค้ดได้ดีที่สุดในปัจจุบัน พร้อมเรียนรู้การแยกส่วนประกอบของแอปพลิเคชัน (Separation of Concerns) การเชื่อมต่อ API ขั้นสูง การจัดการข้อมูลแบบ Asynchronous และการทำ Routing เพื่อให้สามารถนำความรู้ไปสร้างแอปพลิเคชันระดับองค์กรที่ง่ายต่อการทดสอบ บำรุงรักษา และพร้อมรองรับการเติบโตในอนาคตได้อย่างแท้จริง

## วัตถุประสงค์ของหลักสูตร

- เพื่อให้ผู้เข้าอบรมเข้าใจและสามารถออกแบบสถาปัตยกรรมแอปพลิเคชัน (Scalable Architecture) ที่รองรับการขยายตัวในอนาคตได้อย่างถูกต้อง
- เพื่อให้สามารถใช้งาน Riverpod สำหรับการจัดการ State ทุกรูปแบบ ตั้งแต่ระดับพื้นฐานไปจนถึงระดับซับซ้อนได้อย่างมีประสิทธิภาพ
- เพื่อให้สามารถแยกส่วนการทำงาน (Separation of Concerns) ระหว่าง UI Layer, Business Logic Layer และ Data Layer ได้อย่างชัดเจน
- เพื่อพัฒนาทักษะการจัดการข้อมูลแบบ Asynchronous, การทำ Caching และการเชื่อมต่อ REST API ผ่าน Riverpod และ Dio
- เพื่อยกระดับการเขียนโค้ดให้เป็น Clean Code ที่ง่ายต่อการอ่าน การทำ Unit Testing และการบำรุงรักษาในระยะยาว
- เพื่อให้ผู้เรียนสามารถนำหลักการ Mobile Security มาใช้จริง ได้แก่ SSL Pinning, Data Encryption และการจัดเก็บข้อมูลลับอย่างปลอดภัย
- เพื่อให้สามารถ Optimize Performance ของแอป ลด Jank, จัดการ Memory และวัดผลด้วย Flutter DevTools ได้อย่างเป็นระบบ
- เพื่อให้เข้าใจการเชื่อมต่อ Native Platform ผ่าน Platform Channel และสามารถออกแบบ Production-ready App ได้ครบถ้วน

# Day 1: Scalable Architecture & Riverpod Fundamentals

## เป้าหมายของวันแรก

ผู้เรียนจะเข้าใจหลักการของ Scalable Architecture สมัยใหม่ วางโครงสร้างโปรเจกต์ระดับ Enterprise ได้อย่างถูกต้อง และเริ่มต้นใช้งาน Riverpod เป็นเครื่องมือจัดการ State หลักของแอปพลิเคชัน

### Introduction to Modern Flutter Architecture (2026)

- แนวคิด Clean Architecture และโครงสร้างโปรเจกต์ระดับ Enterprise
- การจัดโครงสร้างโฟลเดอร์แบบ Feature-first vs Layer-first และข้อดี-ข้อเสียของแต่ละแบบ

### Deep Dive into Riverpod

- ทำไมต้อง Riverpod? ข้อดีและวิวัฒนาการจาก Provider
- การติดตั้งและการตั้งค่า ProviderScope

### Understanding Core Providers

- การใช้งาน Provider, StateProvider และ FutureProvider
- อัปเดตล่าสุด: การเปลี่ยนผ่านไปสู่ Notifier และ AsyncNotifier

### Consuming State

- การใช้ ConsumerWidget และ ConsumerStatefulWidget
- ความแตกต่างและการเลือกใช้ ref.watch(), ref.read() และ ref.listen()

**Workshop Day 1 (BLA Policy Companion):** วางโครงสร้างโปรเจกต์แบบ Feature-first และจัดการ State หน้ารายการกรมธรรม์เบื้องต้นด้วย Riverpod

# Day 2: Advanced Data Handling & Dependency Injection

## เป้าหมายของวันที่สอง

ผู้เรียนจะจัดการข้อมูลแบบ Asynchronous ได้อย่างมืออาชีพ ทำระบบ Caching และ Optimization พร้อมเชื่อมต่อ API ขั้นสูง และเข้าใจการทำ Dependency Injection ด้วย Riverpod

### Asynchronous Data Management

- การจัดการสถานะ Loading, Data และ Error อย่างสง่างามด้วย AsyncValue
- การใช้ AsyncNotifierProvider สำหรับการดึงข้อมูล (Fetching) และปรับปรุงข้อมูล (Mutation)

### Smart Caching & Optimization

- การทำ Data Caching, Polling และ Auto-refresh ข้อมูลเมื่อมีอัปเดต
- การใช้คุณสมบัติ keepAlive และการทำลาย State ที่ไม่ใช้งาน (Dispose)

### Dependency Injection (DI) with Riverpod

- การฉีด Dependencies ให้กับ Repository และ Service Layer แบบไม่ต้องพึ่งพา Package อื่น

### Advanced API Integration

- การเชื่อมต่อ REST API ขั้นสูงร่วมกับ Dio (การทำ Interceptors และ Error Handling)

### Skeleton Loading, Shimmer UI & Lazy Loading

- หลักการออกแบบ Loading State ที่ดีสำหรับ Production App — จาก Loading Spinner สู่ Skeleton Screen
- การสร้าง Shimmer Effect และ Skeleton UI ด้วย `shimmer` / `skeletonizer` Package เพื่อประสบการณ์ผู้ใช้ที่ราบรื่น
- แนวทาง Lazy Loading รายการข้อมูลขนาดใหญ่ด้วย `ListView.builder`, Pagination และ Infinite Scroll ร่วมกับ Riverpod

**Workshop Day 2 (BLA Policy Companion):** สร้างระบบดึงข้อมูลกรมธรรม์จาก API ที่มี Caching อัตโนมัติ, Pull-to-refresh, การยื่นคำขอสินไหม (Mutation) และการจัดการ Error State เชิงลึกผ่าน Riverpod

# Day 3: Advanced Routing, Security & Production Readiness

## เป้าหมายของวันที่สาม

ผู้เรียนจะทำ Advanced Routing ที่ควบคุมด้วยสถานะ จัดการความปลอดภัยของแอปพลิเคชัน เขียน Test สำหรับ Riverpod และเข้าใจ Best Practices เพื่อเตรียมแอปสู่ Production

### Advanced Routing with GoRouter & Riverpod

- การตั้งค่า GoRouter พื้นฐาน และการส่งพารามิเตอร์
- การทำ Route Guards เพื่อปกป้องหน้าจอ (เช่น ต้อง Login ก่อนเข้าถึง) โดยใช้สถานะจาก Riverpod

### Application Security

- การจัดเก็บข้อมูลความลับ (Tokens/Credentials) ด้วย Flutter Secure Storage
- การจัดการ Session และ Auth State ของแอปพลิเคชันแบบ Reactive
- **SSL Pinning ใน Flutter:** หลักการ Certificate Pinning / Public Key Pinning, การตั้งค่า Dio ให้ตรวจสอบ Certificate และป้องกันการโจมตีแบบ Man-in-the-Middle (MITM)
- **Data Security & Encryption:** การเข้ารหัส Request/Response payload ด้วย AES, การจัดการ Key อย่างปลอดภัย และแนวทาง Obfuscation สำหรับ Release Build

### Testing in Riverpod

- แนวทางการเขียน Unit Testing สำหรับ Providers และ Notifiers
- การ Mock Dependencies เพื่อการทดสอบที่รวดเร็ว

### Platform Channel — Native Integration

- แนวคิด Platform Channel คืออะไร และทำไมถึงจำเป็นสำหรับ Production App
- การสร้าง `MethodChannel` เพื่อเรียก Native API บน Android (Kotlin) และ iOS (Swift)
- Use Cases จริง เช่น การเข้าถึง Biometric Authentication, Device Info, หรือ Custom Native SDK
- การจัดการ Error และ Async Result ที่ส่งกลับจาก Native ฝั่ง

### Performance Optimization & Best Practices 2026

- **ลด Jank และ Frame Drop:** ทำความเข้าใจ Flutter Rendering Pipeline, การใช้ `const` Widget, `RepaintBoundary` และการวัดประสิทธิภาพด้วย Flutter DevTools (Performance Overlay)
- **Optimize Rendering:** เทคนิค Widget Rebuild ที่ถูกต้อง — ข้อควรระวังในการใช้ `ref.watch` / `select()` เพื่อ Rebuild เฉพาะส่วนที่เปลี่ยน
- **Memory Management:** การจัดการ Memory Leak, การใช้ `keepAlive` อย่างมีสติ, การ Dispose Resources และการตรวจสอบ Memory ด้วย DevTools
- **Production-ready App — Checklist ก่อน Release:** Code Obfuscation, Logging Strategy, Error Monitoring (Crash Reporting), Build Flavor (Dev/Staging/Prod) และ Environment Configuration

**Workshop Day 3 (Master Workshop · BLA Policy Companion):** ประกอบร่างแอปให้สมบูรณ์ สร้างระบบ Authentication (Login/Logout) ที่ควบคุม Route Guard, จัดเก็บ Token ด้วย Secure Storage และเขียน Unit Test ของ Provider พร้อมแสดงผลข้อมูลแบบ Reactive ด้วย Riverpod ตลอดทั้งระบบ

- **ทักษะจัดการข้อมูลขั้นสูง:** ทำ Async Data, Caching, Skeleton Loading และเชื่อม API ด้วย Dio ได้อย่างมั่นใจ
- **Mobile Security ครบวงจร:** ทำ SSL Pinning, Data Encryption, Secure Storage และป้องกัน MITM Attack ได้จริง
- **Performance ระดับ Production:** วินิจฉัยและแก้ Jank, Memory Leak ด้วย Flutter DevTools พร้อม Best Practices ก่อน Release
- **Native Integration:** เชื่อมต่อ Native API ทั้ง iOS และ Android ผ่าน Platform Channel ได้อย่างถูกต้อง
- **Clean Code ที่บำรุงรักษาได้:** เขียนโค้ดที่อ่านง่าย ทดสอบง่าย และพร้อมต่อยอดในระยะยาว
- **ประสบการณ์ Workshop จริง:** ไม่ใช่แค่ดูสาธิต แต่ลงมือสร้างโปรเจกต์จำลองปัญหาจริงด้วยตัวเอง
